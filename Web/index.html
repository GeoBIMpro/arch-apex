The Problem

Earning a driver’s license implies competency and safety in a large variety of driving scenarios. It is a skill assessment used to bound the risk of operating a vehicle.

Under what criteria can we determine that an autonomous vehicle is safe? How can we generalize its actions beyond simple tests?

Our Solution

APEX is design-time approach to formally verifying the trajectory planning and trajectory tracking stacks of an ADAS/AV as they interact with potentially dynamic participants in a variety of driving scenarios.

Identify suit of canonical driving scenarios and map scenarios to agents. Compose agents according to formal semantics of hybrid systems. Specify a set of desireable properties for each scenario and formally verify that such properties are satisfied.

Requirements

Operating Systems

Linux and OSX

APEX supports Ubuntu 14.04+ and Mac OSX 10.9.5+

Utilities

Homebrew

It is recommended that OSX users utilize the Homebrew package manager

Compiler

GCC

GCC is used to compile C++ planning stack implementation

Languages

Python

Verification instances of scenarios are compiled from composable hybrid agents using Python

Additional Libraries

Swig

Swig is used to wrap implementation-grade C++ trajectory generation libraries

Armadillo

Armardillo is a C++ linear algebra library used for matrix manipulation

Verification Tools

dReach

APEX currently implements driving scenarios as hyrbid automata and checks model properties using dReach. The lastest dReach binaries are included as part of the APEX source

Installation

GCC

UBUNTU: sudo apt-get install build-essential

Python

OSX: Should already be there… UBUNTU: sudo apt-get install python-dev

Swig

OSX: brew install swig UBUNTU: sudo apt-get install swig

Armadillo

OSX: brew install armadillo UBUNTU: sudo apt-get install libarmadillo-dev

Build Trajectory Generator Library and Wrap in Python

Create interface:

OSX and UBUNTU: swig -c++ -python trajectorygenerator.i

Build library and link armadillo:

OSX and UBUNTU: g++ -fPIC -c trajectorygenerator.cpp -O2 -larmadillo

Build wrapper:

OSX: g++ -fPIC -c trajectorygenerator_wrap.cxx -I/usr/lib/python2.7 -O2 -larmadillo

Note make sure that the Python.h file path is right. usually in most systems you will have to change to pythonX.X/Python.h

UBUNTU: g++ -shared trajectorygenerator.o libtraj_gen_v2.o trajectorygenerator_wrap.o -L/usr/lib/python2.7 -lpython2.7 -o _trajectorygenerator.so -O2 -larmadillo

Build file containing specific function which we will expose to APEX tool:

OSX and UBUNTU: g++ -fPIC -c libtraj_gen_v2.cpp -O2 -larmadillo

Link:

OSX and UBUNTU: g++ -shared trajectorygenerator.o libtraj_gen_v2.o trajectorygenerator_wrap.o -L/usr/lib/python2.7 -lpython2.7 -o _trajectorygenerator.so -O2 -larmadillo

Make APEX.py executable:

OSX and UBUNTU: chmod +x APEX.py

Run APEX:

OSX and UBUNTU: ./APEX.py 3.15.10.02
